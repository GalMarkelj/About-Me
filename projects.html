Project description

Description

With this project I made my first interaction with the back-end, meaning, I create a simple application that interacts with the database trough REST API. I used server-side JavaScript run-time environment called NodeJs with Express framework. Trough user prespective you are able to register, login and then insert your "personal" information and later if you like also update them. With that said, those are all the CRUID operations. I create a local session soo that you can stay logged in even if you refresh the browser or go to another page, but in this case meaning of that session is that if you are logged in you are not able to visit pages login and register.

Setup

You can setup this application locally with instructions I wrote HERE

Registration interface

At registration I'm using express-validator libary ( to check if username has at least 3 characters, if it's a valid email, if the two inserted passwords match and have at least 8 characters ) and for password hashing usgin bcrypt libary.

Login interface

When logging in I'm using passport libary, first for validating login data with user data from the database (cheching if user exists with that username and with bcrypt libary comparing password form the database and inserted password) and if that goes trough, create a local session with passport-local.

Database

I'm using SQLite database on this project and Knex ORM to communicate with the database. In the database we have two entitys connected with foreign key. Main entity is the "User" where we store his username, email and hashed password ( When registering it makes sure that username and email are unique ). Child entity "User_data" stores Users "personal" information.

Content

To display the content form the database I'm using EJS view engien.

Error handling

If something goes wrong while registering or logging in the application makes sure that displays what went wrong soo that the user knows that to do. If something goes wrong in the back every error is displayed in the console.

Admin privileges

I create a separate user called Admin, who has privileges to see all registered users and also delete them by their ID from the database.

Description

When me and my college Luka Štiglic started practical education we were given an assignment to build our first legit website for a customer. First we build the website with raw HTML, CSS and JavaScript as result to get to know better with those languages. Then we build it in Wordpress with WPBakery and Istart theme from anpsthemes. Meaning of that website is that a customer can promotes herself by showing what her business is all about and what it offers.

Development

Most of the content is inserted with Wordpress elements, but when we talk about stylesheet, we set some themes intergrated settings, but mostly all of the stylesheet is custom.

Plugins

The plugins we used :

  • Anps Theme plugin
  • Contact Form 7 - Just another contact form plugin. Simple but flexible.
  • Envato Market - WordPress Theme & Plugin management for the Envato Market.
  • Newsletter - Newsletter is a cool plugin to create your own subscriber list, to send newsletters, to build your business. Before update give a look to this page to know what's changed.
  • Safe SVG - Allows SVG uploads into WordPress and sanitizes the SVG before saving it
  • Slider Revolution - Premium responsive slider
  • Smash Balloon Instagram Feed - Display beautifully clean, customizable, and responsive Instagram feeds.
  • Visual Portfolio, Posts & Image Gallery - Modern gallery and portfolio plugin with advanced layouts editor. Clean and powerful gallery styles with enormous settings in the Gutenberg block.
  • WP Responsive Recent Post Slider/Carousel - Add and display Responsive WordPresss Recent Post Slider and Carousel on your website with 4 designs (Slider) and 1 designs (Carousel) using a shortcode. Also added Gutenberg block support.
  • WPBakery Page Builder - Drag and drop page builder for WordPress. Take full control over your WordPress site, build any layout you can imagine – no programming knowledge required.
  • Yoast SEO - The first true all-in-one SEO solution for WordPress, including on-page content analysis, XML sitemaps and much more.

For our requirements these plugins are okey but not exactly what we need. To get what we need we would need to buy premium version of that plugin, so that this did not happen we used those plugins and hardcode them.

Shortcode

For some things that we had to build, plugin don't exist so we had to create a custom shortcode. The good example is the slider to display customer results. You can see how it looks in the top right picutre above. First I wrote raw static markup, but if the customer wanted to change some value, she would had to change it in the markup, so to avoid that I build a custom shortcode. All that customer had to do is to insert values in Wordpress elemetn that I created and the shortcode did the rest. I had skeleton markup written in PHP and the picture and values next to it are all going dynamically from Wordpress element into the markup by a customer.

Overall

With this project I got some knowledge how to set up Wordpress on Linux, build websites with Wordpress, using its plugins and modifying them and building custom shortcodes.
